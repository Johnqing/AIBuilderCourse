# 课程实施配置

## 一、班级分组机制

### 1.1 班级规模

| 班级类型 | 人数 | 小组数 | 每组人数 |
|---------|------|--------|---------|
| 小班 | 12-20人 | 3-4组 | 4-5人 |
| 中班 | 21-30人 | 5-6组 | 4-5人 |
| 大班 | 31-50人 | 7-10组 | 4-5人 |

### 1.2 分组原则

```
分组原则：

1. 异质分组
   - 避免同一部门扎堆
   - 混合技术/非技术背景

2. 角色平衡
   - 每组需有产品/运营/技术角色
   - 确保每组有能落地的业务场景

3. 性格互补
   - 搭配外向/内向
   - 搭配创意型/执行型
```

### 1.3 组长选拔

| 选拔条件 | 职责 |
|---------|------|
| 有项目管理经验 | 进度把控 |
| 有一定技术背景 | 技术答疑 |
| 沟通能力强 | 对外联络 |

## 二、企业配合要求

### 2.1 企业需要提供的支持

| 支持项 | 说明 | 必须/可选 |
|-------|------|---------|
| 学习时间 | 每天4小时脱产学习 | 必须 |
| 业务场景 | 提供真实业务场景 | 必须 |
| 技术支持 | OpenClaw账号、API权限 | 必须 |
| 导师资源 | 安排内部导师跟进 | 推荐 |
| 领导认可 | 项目负责人支持 | 必须 |

### 2.2 企业配合检查清单

- [ ] 确认学员名单
- [ ] 确认脱产学习时间
- [ ] 确认业务场景负责人
- [ ] 开通OpenClaw账号
- [ ] 配置飞书群
- [ ] 安排内部导师

## 三、学习环境配置

### 3.1 账号配置

| 账号 | 数量 | 用途 |
|------|------|------|
| OpenClaw | 每人1个 | 练习使用 |
| 飞书 | 每人1个 | 沟通、通知 |
| Codex/Claude | 每组1个 | AI辅助编程 |

### 3.2 硬件要求

| 设备 | 要求 | 数量 |
|------|------|------|
| 电脑 | 能上网，能运行浏览器 | 每人1台 |
| 投影仪 | 用于讲师演示 | 1台 |
| 白板 | 用于讨论 | 1块 |

### 3.3 空间要求

| 空间 | 要求 | 用途 |
|------|------|------|
| 教室 | 能容纳全班 | 理论课+工作坊 |
| 分组讨论区 | 每组1个区域 | 小组讨论 |
| 茶水间 | 休息、社交 | 课间休息 |
