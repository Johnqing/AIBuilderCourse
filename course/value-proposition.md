# 课程价值主张

> 不教你「什么是AI」，教你「用AI做出东西」，教你「成为能独立完成产品设计和落地的Builder」

---

## 企业培训价值

### 老板关心什么？

| 老板的痛点 | 我们能解决 |
|-----------|-----------|
| 花了钱培训，但看不到效果 | **90天后有可演示的成果**，不是听课记录 |
| 学会了但落不了地 | **用自己公司的真实业务场景**做练习 |
| 培训完人跑了 | **L1-L5等级认证体系**，留住有证书的人才 |
| 不知道员工学得怎么样 | **嵌入式考核**，讲师全程跟踪评估 |

### 学员关心什么？

| 学员的担心 | 我们能解决 |
|-----------|-----------|
| 零基础怕听不懂 | **M0预备模块打基础**，先学后做 |
| 太难了跟不上 | **讲师带练环节**，每步跟做 |
| 学完还是不会用 | **循序渐进练习**，从简单到复杂 |
| 学了用不上 | **作业就是真实工作**，学了就能用 |
| 想创业但不知道怎么落地 | **L5一人公司认证**，带着真实项目来 |

---

## 学员能得到什么？

### 能力层面——Builder能力模型

```
学完90天，你能成为Builder：

├── 提示词工程（M0）
│   └── 从「会用AI」升级到「用好AI」
│
├── PM×AI（M1）
│   └── 用AI辅助需求分析、方案设计、项目管理、数据汇报
│
├── Vibe Coding（M2）
│   └── 用AI写代码、调试、工程化
│
├── 业务分析（M3）
│   └── 识别AI机会、设计方案、测算ROI
│
├── 技术实施（M4）
│   └── 用OpenClaw配置完整工作流
│
└── 实战落地（M5）
    └── 真实场景从0到1，答辩验收
```

### 认证层面——L1-L5认证体系

```
90天可获得：

├── L1认证 - AI Builder操作认证
├── L2认证 - Vibe Coding编程认证
├── L3认证 - 方案设计认证
├── L4认证 - 技术实战认证
└── L5认证 - 一人公司商业落地认证（可选）
```

### 毕业路径

| 路径 | 获得认证 | 适合人群 |
|------|---------|---------|
| 技术路径 | L1+L2+L3+L4 | 想成为技术Builder |
| 综合路径 | L1+L2+L3+L4+L5 | 想成为一人公司创始人 |

## 一句话总结

> **90天后，企业有可用的AI工作流，学员有可证明的Builder能力认证。**

---

## 课程亮点

| 亮点 | 说明 |
|------|------|
| 真实场景 | 用自己公司的业务做练习 |
| 嵌入式考核 | 讲师在授课过程中完成评估，不设额外考试 |
| 能力阶梯 | 从提示词工程→PM×AI→Vibe Coding→商业落地 |
| 一人公司 | L5认证支持创业路径 |
| 班型适配 | 1-50人都能教 |
