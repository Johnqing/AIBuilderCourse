# M2 Vibe Coding 学员每日指南

> Week 3-6：用AI辅助编程，快速构建产品原型
> **核心主题**：工具使用（L1）→ 调试能力（L2）→ 工程化能力（L3）→ 综合能力（L4）
> 每天学习时长：4小时（上午跟练 + 下午独立练习）

---

## 能力阶梯总览

```
L1 工具使用：会用 AI 写代码，Prompt 质量 = 代码质量

L3 工程化能力：代码可维护、可扩展、可协作
L4 综合能力：独立完成完整项目
```

## Week 3：L1 工具使用

### Day 1：AI编程思维 + Codex入门

**今日目标**
- [ ] 理解 vibe coding 的真实面貌（会出错，要调试）
- [ ] 掌握结构化 Prompt 的写法
- [ ] 能用 Codex 生成可运行的代码

**今日任务**

```
跟练任务（上午）：
体验 Prompt 质量差异

Prompt版本1：「写一个待办列表」

Prompt版本2：
「用 HTML+CSS+JavaScript 写一个待办列表，要求：
- 可以添加任务
- 可以删除任务
- 点击任务显示删除线
- 数据保存在 localStorage
- 界面简洁现代」

对比：两版 Prompt 的代码质量差异

独立练习（下午）：
从你的工作中选一个任务，用结构化 Prompt 让 AI 生成代码
记录：Prompt 质量对代码质量的影响
```

**每日复盘**
1. 你的 Prompt 写得够清楚吗？AI 有没有理解偏差？
2. 生成的代码有哪些问题？是 Prompt 没写清楚还是 AI 理解错了？

---

### Day 2：Codex进阶 + 错误预判

**今日目标**
- [ ] 学会「错误预判」——在 Prompt 里堵住漏洞
- [ ] 掌握错误预判三步法
- [ ] 能写出防错的 Prompt

**错误预判三步法**

```
第一步：想清楚 AI 可能在哪里出错
  → 边界条件？非法输入？空值？

第二步：在 Prompt 里明确约束
  → 把防错的逻辑写进去

第三步：让 AI 自己检查
  → 「写完后检查有没有遗漏」
```

**今日任务**

```
跟练任务（上午）：
错误预判练习

1. 用简单 Prompt 生成「文件上传」功能
2. 记录：AI 的代码有哪些问题？

3. 做错误预判，写高质量 Prompt：
「用 HTML+JavaScript 写一个文件上传页面，要求：
- 只允许上传图片（jpg/png/gif）
- 文件大小不超过 5MB
- 空文件提交提示"请选择文件"
- 上传中显示进度条
- 上传成功显示图片预览」

独立练习（下午）：
找一个你之前用 AI 写的代码，分析：如果要堵住可能的 bug，Prompt 应该怎么改？
```

## Week 4：L2 调试能力

### Day 3：代码调试：定位问题

**今日目标**
- [ ] 学会读懂报错信息
- [ ] 能引导 AI 帮你分析问题所在
- [ ] 能明确说出「问题是什么」

**调试 Prompt 模板**

```
【读懂报错】
「这个报错是什么意思？问题在第几行？是什么原因？」

【定位问题】
「问题是：[描述具体问题]
请修复，并说明：
1. 改了哪里
2. 为什么这样改
3. 原来为什么错」
```

**今日任务**

```
跟练任务（上午）：
给一个有 bug 的代码，让 AI 分析

1. 把报错信息发给 AI：「这个报错什么意思？」
2. 根据 AI 分析，说出：「问题是：XXX」
3. 让 AI 修复：「问题是 XXX，请修复」
4. 修完测试，确认修好

跟练任务（下午）：
给一个有 6 个 bug 的代码，一次修一个问题
记录每个 bug 的修复过程
```

---

### Day 4：代码调试：修复问题

**今日目标**
- [ ] 掌握「一次只修一个问题」的原则
- [ ] 掌握「修完必须测试」的铁律
- [ ] 能引导 AI 正确修复，而不是越修越乱

**修复铁律**

```
一次只修一个问题！
修完必须测试！
```

**今日任务**

```
跟练任务（上午+下午）：
给一个有 6 个 bug 的计算器代码

要求：
1. 一次只修一个问题
2. 修完马上测试，确认修好
3. 再修下一个
4. 记录每个问题的修复日志

禁止：
- 让 AI 一次修多个 bug
- 不测试就跑下一个
```

## Week 5：L3 工程化能力

### Day 5：代码重构与代码质量

**今日目标**
- [ ] 能让 AI 帮代码加注释、优化结构
- [ ] 理解重构 = 不改变功能，只改善质量
- [ ] 掌握「小步快走」的重构方式

**重构铁律**

```
每次小改动，动完就测！
重构不改变功能，只改善质量
```

**今日任务**

```
跟练任务（上午）：
给一段「能跑但很乱」的代码，让 AI 重构

1. 「给这段代码加注释，说明每个函数的作用」
2. 「把变量名改得更清晰」
3. 「优化代码结构」
4. 「检查有没有性能问题」

每次改动后测试，确保功能不变。

独立练习（下午）：
找一个之前写的代码，用重构 Prompt 逐步优化
记录每一步的改动和测试结果
```

### Day 6：项目集成与接口设计

**今日目标**
- [ ] 学会先定义接口规范，再分别开发
- [ ] 能解决前后端数据格式不对齐的问题
- [ ] 能让 AI 帮你写 API 文档

**接口对接三步法**

```
第一步：先让 AI 写 API 文档
第二步：前后端各自按规范开发
第三步：对接时先测接口格式
```

**今日任务**

```
跟练任务（上午+下午）：
做前后端分离的 TODO 应用

1. 先让 AI 写 API 文档：
「帮我写 TODO 应用的 API 文档，包含：
- GET /api/todos
- POST /api/todos
- DELETE /api/todos/:id
每个说明：请求格式、响应格式、字段含义」

2. 基于文档生成后端（Express + Node.js）

3. 基于文档生成前端（HTML + JavaScript）

4. 对接测试：如果格式不对齐，用调试方法解决
```

### Day 9-10：项目答辩

**答辩重点**

```
答辩不是展示「代码多漂亮」
而是展示「遇到问题时，你是怎么解决的」

必须回答的问题：
1. 项目遇到了哪些错误/问题？
2. 你怎么分析这个问题的？
3. 你让 AI 怎么修的？
4. 这个过程你学到了什么？
```

**评审标准**

| 维度 | 占比 | 重点 |
|------|------|------|
| 问题解决能力 | 30% | 分析思路 |
| 代码质量 | 25% | 可运行、结构清晰 |
| AI使用效率 | 20% | Prompt 结构化 |
| 调试过程 | 15% | 有问题解决日志 |
| 演示表现 | 10% | 表达清晰 |

## 常见问题 FAQ

| 问题 | 答案 |
|------|------|
| 报错信息看不懂 | 把报错发给 AI：「这个报错什么意思？」 |
| AI 修完还报错 | 让 AI 分析：「改了哪里？还可能有什么问题？」 |
| 一次让 AI 修太多出错 | 一次只修一个问题，修完测试，再修下一个 |
| 对接时格式不对 | 先查接口文档，让 AI 帮你对齐两边格式 |
| 怎么判断重构完成 | 功能不变、代码可读性提升、没有明显性能问题 |
