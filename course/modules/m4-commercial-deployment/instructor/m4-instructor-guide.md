## 核心理念：M3 vs M4 的分工

```
M3 = 人做方案（设计方案、测算ROI）
M4 = 把方案实现（用OpenClaw配置工作流）

M4讲师的核心任务：
不是讲AI能做什么，而是让学员动手配置工作流。

M4 vs M1 的区别：
- M1：用AI Builder提升PM个人效率（发周报、催进度）
- M4：把AI解决方案商业化落地（完整工作流+测试验证）

M4产出：
不是"节省了多少时间"，而是"配置出了可运行的工作流"。
```

## M4-1 AI客服工作流配置（Week 9 Day 1）

### 课前准备
- [ ] 准备AI客服演示环境（意图识别+知识库）
- [ ] 准备意图识别配置截图
- [ ] 准备知识库结构模板

### 理论逐字稿（30min）

```
今天进入M4。

M3你们设计了AI客服方案——
今天要把这个方案用OpenClaw配置出来。

配置AI客服工作流，有几个关键节点：

第一，消息入口
用户从飞书/企微/网页发消息，触发工作流。

第二，意图识别
这是AI客服的核心——让AI理解用户想问什么。
配置要点：
- 定义几个意图（退货/咨询/投诉/其他）
- 每个意图提供10+条训练数据
- 设置置信度阈值（建议80%）

第三，知识库检索
AI根据意图，到知识库找答案。
知识库要提前维护，按意图分类。

第四，路由分支
高置信→AI直接回复
中置信→AI回复+提示转人工
低置信→转人工

这就是AI客服的配置逻辑。
M3你们设计了方案，今天动手把它配置出来。
```

### 跟练任务（90min）

```
任务：配置AI客服工作流

1. 创建工作流「AI客服-姓名」
2. 配置消息入口节点（飞书）
3. 配置意图识别节点（4个意图+训练数据）
4. 配置知识库检索节点
5. 配置路由分支（3个置信度档位）
6. 配置AI回复节点
7. 配置转人工节点
8. 手动触发，测试闭环
```

## M4-2 内容自动化配置（Week 9 Day 2）

### 理论逐字稿（30min）

```
今天配置内容自动化工作流。

链路：选题 → AI生成 → 质量审核 → 人工确认 → 发布

关键节点：

质量审核节点：
AI生成内容后，自己先打分。
- 原创度≥70%
- 相关性≥60%
- 完整度≥80%

低于阈值的，进人工审核。
高于阈值的，进发布流程。

人工确认节点：
AI生成的内容，重要的一定要人确认。
配置：人工确认通过→发布，拒绝→打回重新生成。
```

### 跟练任务（90min）

```
任务：配置内容自动化工作流

1. 创建工作流「内容自动化-姓名」
2. 配置定时触发（每天9点）
3. 配置AI生成节点
4. 配置质量审核节点（3个维度+阈值）
5. 配置分支：达标/不达标
6. 配置人工确认节点
7. 配置发布节点
8. 手动触发，测试
```

## M4-3 销售自动化配置（Week 10 Day 1）

### 理论逐字稿（30min）

```
今天配置销售自动化工作流。

核心是线索打分：
当CRM有新线索进入，AI自动打分，判断意向度。

打分维度：
- 行为数据40%：访问次数、停留时长、下载资料
- 属性匹配30%：行业、职位、公司规模
- 互动数据20%：打开邮件、点击链接、报名活动
- 渠道来源10%：官网、公众号、广告投放

配置Webhook触发：
CRM系统有新数据 → 发Webhook到OpenClaw → 触发工作流

打分后：
- 80分以上 → 立即通知销售
- 60-80分 → 定时汇总通知
- 60分以下 → 培育流程
```

### 跟练任务（90min）

```
任务：配置销售自动化工作流

1. 创建工作流「销售自动化-姓名」
2. 配置Webhook触发节点
3. 配置线索打分节点（4个维度）
4. 配置条件分支（80分/60分/其他）
5. 配置多级通知节点
6. 用模拟CRM数据测试
```

## M4-4 数据监控自动化配置（Week 10 Day 2）

### 理论逐字稿（30min）

```
今天配置数据监控自动化。

定时拉取数据 → AI判断异常 → 多级告警

三个告警级别：
- 黄色：指标变化>5% → 飞书群通知
- 橙色：指标变化>10% → @负责人
- 红色：指标变化>20% → @负责人+抄送上级

定时触发：
每天下午5点自动跑一遍
数据拉到工作流 → AI判断 → 异常则告警

这个工作流的价值：
不是等人在周五发现问题，
而是AI每天监控，周五之前就把异常抓出来。
```

### 跟练任务（90min）

```
任务：配置数据监控工作流

1. 创建工作流「数据监控-姓名」
2. 配置定时触发（每天下午5点）
3. 配置数据拉取节点（模拟数据）
4. 配置异常判断节点（3个级别）
5. 配置多级告警节点
6. 用正常数据和异常数据分别测试
```
