# M4 商业落地

## M4-1 AI客服工作流配置（Week 9 Day 1）

### 学习目标
- [ ] 能把M3设计的AI客服方案，用OpenClaw配置出来
- [ ] 掌握意图识别、知识库、路由分支的配置方法
- [ ] 能处理配置中的常见问题

### 理论内容（30min）

#### M3方案 vs M4配置

```
M3方案设计：
- 用户旅程地图
- 转人工触发条件
- MVP功能定义
- ROI测算

M4技术实施：
- 用OpenClaw把方案配置出来
- 意图识别节点配置
- 知识库关联
- 路由分支设置
- 测试验证闭环
```

#### 核心节点配置

```
意图识别节点：
- 定义意图（退货/咨询/投诉/其他）
- 每意图提供10+条训练数据
- 设置置信度阈值（建议80%）

知识库关联：
- 按意图分类的知识库结构
- QA对格式
- 命中策略（精确/模糊）

路由分支：
- 置信度 > 80% → AI直接回复
- 置信度 60-80% → AI回复 + 提示转人工
- 置信度 < 60% → 转人工
```

### 跟练内容（30min）

```
任务：配置AI客服工作流（M3方案的OpenClaw实现）

1. 创建工作流「AI客服-姓名」
2. 配置消息入口节点（飞书）
3. 配置意图识别节点（定义4个意图）
4. 配置知识库检索节点
5. 配置路由分支（3个置信度档位）
6. 配置AI回复节点
7. 配置转人工节点
8. 保存并测试闭环
```

## M4-2 内容自动化配置（Week 9 Day 2）

### 学习目标
- [ ] 能配置选题→生成→审核→发布全链路
- [ ] 能设置质量评分节点
- [ ] 能配置人工确认节点

### 理论内容（30min）

#### 内容自动化链路

```
选题 → AI生成 → 质量审核 → 人工确认 → 发布
              ↓
         质量不达标 → 打回修改
```

#### 质量评分配置

```yaml
质量评分维度：
- 原创度：≥70%（AI自检）
- 相关性：≥60%（关键词匹配）
- 完整度：≥80%（要素齐全）

触发规则：
- 评分 ≥ 阈值 → 自动进入发布流程
- 评分 < 阈值 → 发送通知给人工审核
```

#### 人工确认节点

```
为什么要人工确认：
- AI生成的内容可能出错
- 涉及品牌调性需要人把关
- 敏感内容需要人工审核

配置要点：
- 人工确认通过 → 继续发布
- 人工确认不通过 → 打回给AI重新生成
```

### 跟练内容（30min）

```
任务：配置内容自动化工作流

1. 创建工作流「内容自动化-姓名」
2. 配置定时触发（每天9点选题）
3. 配置AI生成节点（生成5条文案）
4. 配置质量审核节点（3个维度）
5. 配置分支：
   - 达标 → 发通知进入发布
   - 不达标 → 发通知人工审核
6. 配置人工确认节点
7. 保存并测试
```

## M4-3 销售自动化配置（Week 10 Day 1）

### 学习目标
- [ ] 能配置线索打分节点
- [ ] 能配置多级触发提醒
- [ ] 理解Webhook数据对接

### 理论内容（30min）

#### 线索打分配置

```yaml
打分维度：
- 行为数据（40%）：访问次数、停留时长、下载资料
- 属性匹配（30%）：行业、职位、公司规模
- 互动数据（20%）：打开邮件、点击链接、报名活动
- 渠道来源（10%）：官网、公众号、广告投放

触发规则：
- 80分以上 → 高意向，立即企微通知销售
- 60-80分 → 中意向，每天定时汇总
- 60分以下 → 低意向，进入培育流程
```

#### Webhook数据对接

```
CRM系统 → Webhook → OpenClaw工作流

Webhook触发配置：
- 接收URL：工作流的Webhook地址
- 数据格式：JSON
- 字段映射：从CRM字段 → OpenClaw变量
```

### 跟练内容（30min）

```
任务：配置销售自动化工作流

1. 创建工作流「销售自动化-姓名」
2. 配置Webhook触发节点
3. 配置线索打分节点（4个维度）
4. 配置条件分支（80分/60分/其他）
5. 配置通知节点（企微/飞书/邮件）
6. 保存并用模拟数据测试
```

## M4-4 数据监控自动化配置（Week 10 Day 2）

### 学习目标
- [ ] 能配置定时数据拉取
- [ ] 能配置异常判断分支
- [ ] 能配置多级告警通知

### 理论内容（30min）

#### 数据监控链路

```
定时触发 → 数据拉取 → 异常判断 → 多级告警
                            ↓
                      正常 → 静默/记录
                      异常 → 按级别通知
```

#### 异常判断配置

```yaml
监控指标：
- DAU/MAU
- 转化率
- 响应时间
- 错误率

告警级别：
- 黄色预警：指标变化 > 5%
- 橙色预警：指标变化 > 10%
- 红色预警：指标变化 > 20% 或 绝对值超阈值

通知策略：
- 黄色 → 飞书群通知
- 橙色 → 飞书@负责人
- 红色 → 飞书@负责人 + 邮件抄送上级
```

### 跟练内容（30min）

```
任务：配置数据监控自动化工作流

1. 创建工作流「数据监控-姓名」
2. 配置定时触发（每天下午5点）
3. 配置数据拉取节点（模拟数据）
4. 配置异常判断节点（3个级别）
5. 配置多级告警节点（黄/橙/红）
6. 保存并测试异常分支
```

## M4-5 五大行业案例实施（Week 10 Day 3-5）

### 案例一：教育培训 - 工单智能分发

**场景**：工单系统 → AI识别意图 → 分发给对应顾问

**工作流配置要点**：
- 触发：工单系统Webhook
- 意图识别：课程咨询/报名缴费/售后服务/投诉
- 分发规则：按顾问忙闲状态 + 擅长领域
- 兜底：转人工或留言

**核心指标**：响应45min→3min，人效+57%

---

### 案例二：电商 - 售后工单自动化

**场景**：用户提交售后 → AI自动分类 → 按类型分流

**工作流配置要点**：
- 触发：用户提交售后表单
- AI分类：退货/换货/维修/赔偿
- 分支处理：每类配置不同流程
- 兜底：高价值/差评预警 → 转人工

**核心指标**：自动分流率85%，人效+167%

---

### 案例三：制造业 - 产线报工与质检

**场景**：产线报工 → AI质检判断 → 异常预警 → 班组长处理

**工作流配置要点**：
- 触发：产线系统报工数据
- AI质检：参数比对 + 异常模式识别
- 分支：正常 → 记录；异常 → 预警
- 通知：按车间/班组长发飞书消息

**核心指标**：人效+40%，异常响应-65%

---

### 案例四：医疗机构 - 智能预约分诊

**场景**：患者预约 → AI理解症状 → 推荐科室 → 发送提醒

**工作流配置要点**：
- 触发：患者预约请求
- AI理解：症状描述 → 可能疾病 → 推荐科室
- 知识库：科室擅长+医生排班
- 通知：确认短信/飞书通知患者

**核心指标**：等待时间-60%，预约取消-35%

---

### 案例五：金融 - 贷款材料智能初筛

**场景**：材料提交 → AI初筛 → 完整性检查 → 风险提示 → 人工复核

**工作流配置要点**：
- 触发：客户提交贷款材料
- AI初筛：字段完整性检查 + 格式校验
- 风险提示：AI识别潜在风险点
- 分支：完整无风险 → 快速通过；有问题 → 通知人工

**核心指标**：审批时效+3倍，材料漏交-70%
