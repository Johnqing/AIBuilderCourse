# M1 AI Builder基础 学员每日指南

> Week 1-2：产品经理综合能力提升——用 AI Builder 提升产品经理的核心工作效能
> **核心主题**：产品经理的综合能力——需求分析、方案设计、项目管理、数据汇报
> **工具支撑**：用 AI Builder 把这些能力从「人工」变成「AI辅助」

## 核心理念：AI辅助PM工作

```
产品经理的核心工作：
需求分析 → 方案设计 → 项目管理 → 数据汇报

AI Builder能帮你做的：
把重复性高、价值有限的工作自动化
→ 访谈记录AI总结
→ PRD框架AI生成
→ 周报AI自动汇总
→ 数据异常AI告警

产品经理应该做的（AI替代不了的）：
需求价值判断
优先级决策
跨部门沟通谈判
老板汇报

学习目标：
学会用AI Builder，把自己的日常工作自动化
让自己有更多时间做「需要人判断」的事
```

## Week 1

### Day 1：AI辅助需求分析

**今日目标**
- [ ] 理解产品经理工作中，哪些可以交给AI
- [ ] 学会用AI辅助需求分析：记录→整理→优先级
- [ ] 完成一个AI辅助需求分析的工作流配置

**今日任务**

```
跟练任务（上午）：
场景：用户访谈——"APP打开太慢，希望加功能"

1. 把原始反馈发给AI，让AI分析：
Prompt：「分析这段用户反馈：
1. 核心需求是什么？
2. 背后可能的需求是什么？
3. 如何转化为产品需求？」

2. 让AI用结构化格式整理需求（MoSCoW）

独立练习（下午）：
从你的工作中找一个真实需求，用AI辅助分析：
1. AI总结原始反馈
2. AI给出结构化需求
3. AI建议优先级
4. 对比：AI分析 vs 你自己分析，哪个更准确？
```

**每日复盘**
1. 哪些需求分析环节AI帮到了你？
2. 哪些环节AI还是做不好，必须人来做？

### Day 2：AI辅助方案设计

**今日目标**
- [ ] 学会用AI辅助写PRD和方案文档
- [ ] 能用AI生成方案框架和补充细节
- [ ] 理解AI写方案的边界——框架可以，核心判断还是要人

**今日任务**

```
跟练任务（上午）：
让AI辅助写PRD

Prompt：「帮我写一个[功能名]的PRD框架，包含：背景、目标、用户故事、功能列表、非功能需求、验收标准」

然后补充细节：
Prompt：「帮我补充这个PRD的详细内容：
- 3个典型用户故事
- 5个核心功能点描述
- 异常流程说明」

独立练习（下午）：
选一个你实际工作中的需求，用AI写一个完整PRD，
评估：AI写的PRD和你平时写的，有什么差距？
```

## AI辅助PM工作场景库

### 需求分析阶段

```
【访谈记录总结】
触发：访谈结束
AI操作：录音→文字→结构化总结
输出：发给产品经理审核

【需求文档撰写】
触发：产品经理输入原始需求
AI操作：生成PRD框架+补充细节
输出：PRD草稿，产品经理修改定稿

【需求优先级分析】
触发：需求列表输入
AI操作：按MoSCoW分析+给出建议
输出：优先级建议，产品经理拍板
```

### 方案设计阶段

```
【PRD框架生成】
触发：功能名称输入
AI操作：生成PRD框架
输出：草稿，产品经理补充细节

【方案完整性检查】
触发：方案文档输入
AI操作：检查是否完整
输出：检查结果和改进建议
```

### 项目管理阶段

```
【每日站会提醒】
触发：定时（每天早上9点）
AI操作：发消息催进度→汇总回复→发项目群
输出：每日站会汇总

【周报自动生成】
触发：定时（每周五下午5点）
AI操作：汇总各方进度更新→生成周报草稿
输出：周报草稿，产品经理审核发出

【风险预警】
触发：项目指标超过阈值
AI操作：立即告警到项目群
输出：预警消息
```

### 数据汇报阶段

```
【日报自动生成】
触发：定时（每天下午5点）
AI操作：拉取数据→生成日报→发飞书
输出：日报草稿

【数据异常告警】
触发：指标变化超过阈值
AI操作：立即预警
输出：告警消息

【月度汇报辅助】
触发：每月末
AI操作：整理核心数据→生成汇报框架
输出：汇报材料草稿
```

## 产品经理用AI Builder checklist

```
确定要自动化的场景：
□ 这个工作重复性高吗？
□ 每次做这个工作需要多少时间？
□ 自动化后能省多少时间？

设计工作流时：
□ 触发条件是什么？（定时/事件）
□ AI处理的边界在哪？（什么让AI做，什么让人做）
□ 异常情况怎么处理？

上线前测试：
□ AI输出的质量是否可接受？
□ 人需要审核哪些环节？
□ 发现问题怎么迭代？
```

## 常见问题 FAQ

| 问题 | 答案 |
|------|------|
| AI写的PRD能用吗 | AI写框架可以，核心判断（做什么不做什么）要人定 |
| AI生成的数据报告准吗 | AI整理数据可以，但分析洞察要人做 |
| 自动化后产品经理做什么 | 去做「需要人判断」的事：优先级决策、跨部门沟通、老板汇报 |
| 什么工作不适合自动化 | 非标准化、涉及多方利益、需要人判断的工作不适合 |
