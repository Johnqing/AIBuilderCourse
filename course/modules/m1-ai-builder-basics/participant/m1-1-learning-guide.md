## 课堂学习

### 理论要点

#### 1. AI Builder是什么

**一句话定义**：
AI Builder = AI能力的「操作系统」，让AI能力可以像搭积木一样串联起来。

**类比**：
```
传统开发：写代码 → 调用API → 部署服务
AI Builder：选技能 → 连节点 → 配置工作流
```

#### 2. OpenClaw架构

```
┌─────────────────────────────────────────┐
│              OpenClaw                    │
│                                          │
│  ┌──────────┐      ┌────────────────┐   │
│  │  消息通道  │      │   AI Builder   │   │
│  │ 飞书/企微 │ ←→  │  技能市场      │   │
│  │ 钉钉/Slack│      │  工作流编排     │   │
│  └──────────┘      └────────────────┘   │
│                                          │
│  ┌──────────┐      ┌────────────────┐   │
│  │  定时触发  │ ←→  │  节点管理      │   │
│  │  Webhook │      │  通道集成      │   │
│  └──────────┘      └────────────────┘   │
└─────────────────────────────────────────┘
```

#### 3. 四大核心模块

| 模块 | 作用 | 类比 |
|------|------|------|
| **技能** | AI能力的原子单元 | 乐高零件 |
| **工作流** | 多个技能的编排 | 乐高图纸 |
| **节点** | 工作流中的处理单元 | 流水线工位 |
| **通道** | 消息的入口和出口 | 流水线进出口 |

#### 4. AI Builder能做什么

| 场景类型 | 具体例子 |
|---------|---------|
| 客服自动化 | 智能回复 → 意图识别 → 工单创建 → 分配坐席 |
| 内容自动化 | 选题 → AI生成 → 审核 → 发布 |
| 销售自动化 | 线索打分 → 达标提醒 → 自动外呼 |
| 管理自动化 | 任务分配 → 进度追踪 → 风险预警 |

## 知识卡片

### 卡片1：什么是技能

**定义**：技能是AI能力的原子单元，可以独立完成某个特定任务。

**示例**：
- `feishu-im-read` - 读取飞书消息
- `web-search` - 网络搜索
- `codex` - AI代码生成

**特点**：
- 可单独使用
- 可组合使用
- 可在技能市场找到

---

### 卡片2：什么是工作流

**定义**：工作流是多个节点组成的处理流水线。

**组成**：
```
触发节点 → 处理节点1 → 处理节点2 → 输出节点
```

**示例**：
```
消息入口 → AI理解 → 知识库检索 → AI回复 → 发送消息
```

---

### 卡片3：节点类型

| 节点类型 | 颜色 | 作用 | 示例 |
|---------|------|------|------|
| 触发节点 | 绿色 | 启动工作流 | 消息入口、定时触发 |
| 处理节点 | 蓝色 | 执行具体操作 | AI对话、技能调用 |
| 输出节点 | 紫色 | 输出结果 | 发送消息、写入数据 |

## 跟练任务

### 任务1：熟悉界面（10分钟）

```
1. 登录OpenClaw
2. 找到AI Builder入口（左侧菜单）
3. 点击「技能市场」
4. 记录3个你感兴趣的技能名称
5. 截图发到班级群
```

### 任务2：创建第一个工作流（15分钟）

```
1. 点击「工作流」→「创建工作流」
2. 命名为「test-你的姓名」
3. 从左侧拖入「消息入口」节点
4. 从左侧拖入「AI回复」节点
5. 把两个节点连起来
6. 点击「保存」
7. 截图发到班级群
```

### 任务3：手绘架构图（5分钟）

```
1. 拿出一张纸
2. 画出OpenClaw架构图
3. 包含：消息通道层、AI Builder层、核心模块
4. 拍照发到班级群
```
