# M1 知识卡片

> AI Builder核心概念速查

---

## 卡片1：AI Builder定位

```
定位：OpenClaw中的AI能力编排平台

作用：让AI能力可以像搭积木一样串联起来

一句话：不是让AI帮你做事，而是让AI按照你的流程做事
```

---

## 卡片2：四大核心模块

| 模块 | 英文 | 作用 | 类比 |
|------|------|------|------|
| 技能 | Skill | AI能力的原子单元 | 乐高零件 |
| 工作流 | Workflow | 多个技能的编排 | 乐高图纸 |
| 节点 | Node | 工作流中的处理单元 | 流水线工位 |
| 通道 | Channel | 消息的入口和出口 | 流水线进出口 |

---

## 卡片3：技能类型

```
┌─────────────────────────────────────────┐
│              技能（Skill）               │
├─────────────────────────────────────────┤

│  自定义技能                            │
│  └── 用户可根据需求创建                   │
└─────────────────────────────────────────┘
```

---

## 卡片4：节点类型

| 节点类型 | 图标颜色 | 作用 | 示例 |
|---------|---------|------|------|
| 触发节点 | 绿色 | 启动工作流 | 消息入口、定时触发、Webhook |
| 处理节点 | 蓝色 | 执行具体操作 | AI对话、技能调用、条件判断 |
| 输出节点 | 紫色 | 输出结果 | 发送消息、写入数据、API返回 |

---

## 卡片5：工作流基本结构

```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│ 触发节点 │ ──→ │ 处理节点 │ ──→ │ 处理节点 │ ──→ │ 输出节点 │
│  (绿色) │     │  (蓝色)  │     │  (蓝色)  │     │  (紫色) │
└─────────┘     └─────────┘     └─────────┘     └─────────┘
     ↑               ↑               ↑
  工作流起点      技能执行        结果输出
```

## 卡片6：通道类型

| 通道 | 用途 |
|------|------|
| 飞书 | 企业内部沟通 |
| 企业微信 | 客户沟通 |
| 钉钉 | 企业内部沟通 |
| Slack | 海外企业沟通 |
| Webhook | 第三方系统对接 |

---

## 卡片7：触发方式

| 触发类型 | 说明 | 示例 |
|---------|------|------|
| 消息触发 | 收到用户消息 | 飞书收到消息 |
| 定时触发 | 按时执行 | 每天9点发送日报 |
| 事件触发 | 特定事件发生 | 工单创建时 |
| Webhook | 第三方系统调用 | 支付完成后 |

---

## 卡片8：适合 vs 不适合

| 适合场景 | 不适合场景 |
|---------|-----------|
| 多AI能力串联 | 简单一次性问答 |
| 条件分支判断 | 纯数据处理（用脚本） |
| 定时/事件触发 | 实时对话（用ChatGPT） |
| 跨平台消息路由 | 复杂逻辑计算 |
