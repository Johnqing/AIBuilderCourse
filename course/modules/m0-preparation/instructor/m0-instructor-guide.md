## 核心理念：M0 不是「学用AI」，是「用好AI」

```
M0的假设：
学员已经会基本操作（注册账号、会对话、会复制粘贴）

M0的任务：
把学员从「会用AI」提升到「用好AI」

核心差距：
会用 = 问一个问题，得到一个答案
用好 = 用正确的方式问，得到可用的答案

三个核心能力：
1. 结构化Prompt → 一次做对
2. 错误预判 → 堵住漏洞
3. 追问优化 → 持续改进
```

## M0-1 结构化Prompt（Week 0 Day 1-2）

### 课前准备
- [ ] 准备「低质量Prompt vs 高质量Prompt」对比案例
- [ ] 准备结构化Prompt公式
- [ ] 准备各场景Prompt模板

### 理论逐字稿（30min）

```
M0不是教你怎么用ChatGPT——这个你们应该都会了。

M0教的是：怎么用好AI。

先问大家一个问题：你们用AI的时候，有没有遇到过：
- AI回答太泛、不具体
- AI理解错了你的意思
- 给了一个根本没法用的答案

这些问题的根源，90%是Prompt不够好。

给大家看两个例子：

低质量Prompt：「帮我写一篇文章」

高质量Prompt：
「你是一个资深产品经理，帮我写一份产品需求文档，
要求：包含背景、目标、用户故事、功能列表、验收标准，
格式规范，语言专业，面向开发团队。」

区别在哪？
第一个：AI随便发挥
第二个：角色明确、任务清晰、要求具体

这就是结构化Prompt。

公式：角色 + 任务 + 要求 + 示例

今天的目标：让你们把「会问」变成「会问得好」。
```

### 课堂节奏

```

```

### 常见问题与应对

| 问题 | 应对 |
|------|------|
| 学员说"Prompt长一点就好用" | 强调：不是越长越好，是结构清晰、信息到位才好 |
| 学员问"角色设定有用吗" | 演示：同样任务，有角色 vs 无角色的效果差异 |
| 学员发现AI有时不按要求走 | 引入下一个主题：错误预判 |

## M0-2 错误预判与约束（Week 0 Day 3-5）

### 核心理念

```
今天的核心：学会「错误预判」——在Prompt里堵住可能的漏洞。

为什么AI会出错？
1. 边界情况没考虑（除零、超长输入、空值）
2. 格式要求没明确（要JSON还是纯文本？）
3. 受众没交代（给老板看 vs 给开发看）

错误预判 = 在问题发生之前，想清楚AI可能在哪里出错，
然后在Prompt里把约束写清楚。
```

### 理论逐字稿（30min）

```
昨天学了结构化Prompt，今天更进一步：学会「错误预判」。

先问：AI为什么会出错？

例子：让AI写一个计算器
Prompt1：「写一个计算器」→ AI生成，可能没有处理除零
Prompt2：「写一个计算器，要求：除以0显示错误、输入非数字显示错误」→ AI就会处理

区别在哪？
Prompt1没有预判错误情况
Prompt2把错误情况提前堵住了

这就是「错误预判」。

错误预判三步：
1. 想清楚AI可能在哪里出错
2. 在Prompt里明确约束
3. 让AI自己检查

错误预判清单：
- 边界条件：空值、超长、超限
- 格式要求：JSON/纯文本/代码
- 受众限定：专业术语/通俗语言
- 安全约束：什么不能生成
```

### 跟练任务（90min）

```
任务：Prompt错误预判练习

场景1：让AI生成用户注册表单
- 低质量Prompt → 记录AI出的错
- 高质量Prompt（加约束）→ 验证是否避开了错误

场景2：让AI分析数据
- 低质量Prompt → 记录AI出的错
- 高质量Prompt（加约束）→ 验证是否避开了错误

每个场景对比记录：
1. 低质量Prompt的问题
2. 补充了哪些约束
3. 高质量Prompt的效果
```

## M0-3 批判对话与追问（Week 1 Day 6-8）

### 核心理念

```
今天的核心：学会「批判对话」——不盲目接受AI的答案。

AI的答案不一定对。
你要能质疑、追问、引导AI给你更好的答案。

两种能力：
1. 批判对话：AI说的对不对？证据在哪？
2. 追问优化：怎么追问能让答案越来越精准？
```

### 理论逐字稿（30min）

```
AI说的都是对的吗？不一定。

AI有几个常见问题：
1. 幻觉：自信地胡说八道
2. 泛化：给一个放之四海皆准的答案
3. 讨好：你想听什么它就说什么

所以你要会「批判对话」。

批判对话三技巧：
1. 质疑：「你为什么这么说？证据在哪？」
2. 验证：「能给我一个具体例子吗？」
3. 换角：「从另一个角度分析呢？」

还有一种能力很重要：追问优化。

好的追问能让AI的答案越来越精准：
1. 方向追问：「你说的A具体指什么？」
2. 深挖追问：「为什么A会导致B？」
3. 验证追问：「这个有数据支持吗？」

记住：你是主人，AI是工具。
不要AI说什么你就信什么。
```

### 跟练任务（90min）

```
任务：批判对话练习

1. 问AI一个有争议的问题
   「AI会取代产品经理吗？」

2. 追问：「你说"会取代"，具体指什么？」

3. 追问：「有什么数据支持这个观点？」

4. 追问：「有没有反面的观点？」

5. 追问：「如果我要应对AI取代，应该怎么做？」

对比：原始回答 vs 多轮追问后的回答质量差异
```

## M0-4~5 场景化Prompt练习（Week 2）

### 核心理念

```
今天的核心：把前三天学的三个能力，用到真实工作场景中。

三个场景：
1. 需求分析Prompt
2. 方案设计Prompt
3. 代码生成Prompt

每个场景都要体现：
- 结构化（角色+任务+要求）
- 错误预判（约束条件）
- 追问优化（多轮迭代）
```

### 场景1：需求分析Prompt

```
公式：角色 + 背景 + 需求 + 要求

示例Prompt：
「你是一个资深产品经理，辅助我做需求分析。

背景：用户反馈我们的APP打开速度慢，希望加更多功能。

请帮我分析：
1. 这个反馈背后的真实需求是什么？
2. 可能的产品机会有哪些？
3. 如何转化为可落地的需求？

要求：
- 给出3个具体的产品机会
- 每个机会说明：目标用户、核心功能、预期价值
- 用结构化格式输出」
```

### 场景2：方案设计Prompt

```
公式：角色 + 方案类型 + 约束 + 输出格式

示例Prompt：
「你是一个B端产品专家，帮我设计一个客服系统方案。

要求：
- 支持多渠道接入（飞书、企微、网页）
- 包含AI客服和人工客服切换
- 要有知识库管理功能
- 估算实施难度和周期

输出：
1. 整体架构图（文字描述）
2. 核心功能模块
3. 实施计划（分阶段）
4. 技术注意点」

检查：这个Prompt有哪些预判约束？（多渠道、切换、知识库）
```

### 场景3：代码生成Prompt

```
公式：任务 + 技术栈 + 功能要求 + 约束

示例Prompt：
「用React写一个待办列表组件，要求：
1. 可以添加、删除、完成待办事项
2. 数据保存在localStorage
3. 组件要支持TS类型
4. 有基本的样式（不要Tailwind）

请分步骤：
1. 先写出组件结构
2. 再补充状态逻辑
3. 最后加上样式」
```

## M0-6~7 AI辅助工作流（Week 2 Day 9-12）

### 核心理念

```
今天的核心：把Prompt模板化，形成工作流。

为什么要模板化？
因为很多任务要重复做——与其每次重新写Prompt，
不如把常用场景的Prompt做成模板。

典型工作流Prompt模板：
1. 输入 → 处理 → 输出
2. 模板 = 固定框架 + 变量
3. 每次用：填变量 → 发给AI → 得到结果
```

### 理论逐字稿（30min）

```
你们工作中有没有这样的场景：

每次开会都要写纪要，每次写纪要都要想结构，
能不能把纪要Prompt模板化？

可以的。

把重复性的工作，变成「填变量 → 发AI → 得结果」的工作流。

工作流Prompt公式：
「作为[角色]，处理[输入]，按照[格式]输出」

例子：会议纪要工作流
「作为产品经理，根据以下会议内容，生成结构化会议纪要。

格式：
## 会议目标
## 关键决策
## Action Items（负责人+截止时间）

内容：[粘贴会议内容]」

每次开会，把内容粘进去，Prompt不用变。
这就是工作流思维。
```

## 讲师资源包

### 结构化Prompt公式

```
角色：你希望AI扮演什么角色？
任务：你要AI做什么？
要求：有什么具体要求？
  - 格式（JSON/纯文本/代码/列表）
  - 受众（专业/通俗/老板/开发）
  - 约束（不能做什么/必须做什么）
  - 示例（有没有参考）

完整示例：
「作为[资深产品经理]，帮我[写一份PRD]，要求：
- [包含背景、目标、用户故事、功能列表]
- [面向开发团队，语言专业]
- [格式规范]
- [参考附件的模板风格]」
```

### 错误预判清单

```
写Prompt前，检查：
□ 边界条件：空值、超长输入、非法格式
□ 格式要求：JSON/纯文本/代码/表格
□ 受众限定：专业术语/通俗语言/管理层/执行层
□ 安全约束：不要生成什么
□ 完成标准：什么算完成
```

### 追问优化话术

```
【质疑】
「你为什么这么说？」
「这个有依据吗？」
「有什么数据支持？」

【深挖】
「具体指什么？」
「为什么会这样？」
「A和B是什么关系？」

【多角度】
「从另一个角度呢？」
「有没有反例？」
「如果我是用户，会怎么想？」

【验证】
「这个能落地吗？」
「执行步骤是什么？」
「有什么风险？」
```

### 常见问题 FAQ

| 问题 | 答案 |
|------|------|
| 角色设定有用吗 | 有用，角色决定了AI的专业程度和表达风格 |
| Prompt越长越好吗 | 不是越长越好，是结构清晰、信息到位 |
| AI总是不按要求做 | 检查：是否把约束写清楚了？是否给了示例？ |
| 同一个Prompt每次结果不一样 | 正常，AI有随机性。可以用温度参数或要求AI先复述你的要求 |
