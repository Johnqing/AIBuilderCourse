# M3 场景设计

> Week 7-8：挖掘业务场景，设计AI落地方案
> **核心主题**：业务分析能力——识别AI机会、设计方案、测算价值

---

## 本模块核心目标

```
M3 的定位：业务分析能力

M3 vs M4 的核心区别：
- M3 = 人做方案（需求分析、方案设计、ROI测算）
- M4 = 工具实现（M3方案 → OpenClaw配置）

学完M3，你将能：
- 用AI场景四要素识别AI机会
- 用需求五步法做业务挖掘
- 用三层拆解法把业务问题转化为AI需求
- 用MoSCoW法则排优先级
- 设计完整的AI闭环
- 测算ROI和回本周期
```

## 📖 课程内容

### M3-1：业务挖掘

| 维度 | 内容 |
|------|------|
| 主题 | AI场景四要素、需求访谈五步法、五个行业AI机会 |
| 时长 | 理论30min + 带练30min |
| 核心 | 识别什么业务适合用AI |

### M3-2：需求分析

| 维度 | 内容 |
|------|------|
| 主题 | 业务问题→AI需求三层拆解、MoSCoW法则 |
| 时长 | 理论30min + 带练30min |
| 核心 | 把业务需求转化为可落地的AI需求 |

### M3-3：方案设计

| 维度 | 内容 |
|------|------|
| 主题 | AI闭环设计、用户旅程地图、MVP定义 |
| 时长 | 理论30min + 带练30min |
| 核心 | 设计完整的AI工作流方案 |

### M3-4：成本测算

| 维度 | 内容 |
|------|------|
| 主题 | Token计算、ROI公式、成本收益分析 |
| 时长 | 理论30min + 带练30min |
| 核心 | 算清楚能不能赚钱 |

### M3-5：方案评审

| 维度 | 内容 |
|------|------|
| 主题 | 小组汇报、评委评审 |
| 时长 | 3天 |
