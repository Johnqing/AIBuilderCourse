## AI场景四要素 Checklist

```
适合AI的场景（满足3条以上优先做）：
□ 重复性高：同一问题/操作每天发生多次
□ 规则明确：有明确处理逻辑或足够历史数据
□ 数据充足：有历史记录可供AI学习
□ 价值明显：解决问题能带来可量化收益
□ 技术可行：当前AI能力能支持

不适合AI的场景（优先排除）：
□ 需要深度情感理解
□ 涉及法律合规判断
□ 实时性要求极高
□ 完全无历史数据的全新场景
```

### Day 2：需求分析

**今日目标**
- [ ] 掌握三层拆解法（业务→AI需求→功能）
- [ ] 掌握MoSCoW法则
- [ ] 输出完整的AI需求文档

**三层拆解公式**

```
业务问题：现状中的痛点（一句说清楚）
   ↓
AI需求：AI能做什么（输入→处理→输出）
   ↓
功能要求：具体功能点（MoSCoW）

示例：
业务问题：客服每天回复1000条重复问题，占60%时间
AI需求：需要能自动回复常见问题的AI客服
功能要求：
  - Must：意图识别、知识库检索、AI回复
  - Should：转人工分流
  - Could：满意度调查
```

**今日任务**

```
跟练任务（上午）：
TOP 1 AI机会 → 完整需求文档

1. 用三层拆解法分析
2. MoSCoW排优先级
3. 输出完整需求文档

独立练习（下午）：
另一个AI机会，同样流程做一遍
```

## 常见问题 FAQ

| 问题 | 答案 |
|------|------|
| 找不到合适的AI场景 | 从工作中重复性最高的事开始想 |
| MoSCoW怎么分才合理 | Must不超过60%，核心功能优先 |
| ROI算出来回本要2年 | 可以做，但要和领导说明长期价值 |
| 四要素判断有冲突 | 优先看价值明显+技术可行，重复性可以用人工补 |
