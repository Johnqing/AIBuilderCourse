## 核心理念：M3 vs M4 的分工

```
M3 = 人做方案
M4 = 把方案实现

M3讲师的核心任务：
让学员学会「分析业务 → 设计AI方案 → 测算ROI」
而不是配置工具。

M4讲师的核心任务：
让学员把M3设计的方案，用OpenClaw配置出来。

两个模块的区分：
M3产出：方案文档、ROI数据
M4产出：可运行的工作流

不能让M3变成「配置课」，那是M4的事。
```

## M3-1 业务挖掘（Week 7 Day 1）

### 课前准备
- [ ] 准备AI场景四要素讲解材料
- [ ] 准备需求访谈五步法模板
- [ ] 准备五个行业案例（教育/电商/制造/医疗/金融）

### 理论逐字稿（30min）

```
今天进入M3。

M1你们学了怎么用AI辅助PM日常工作——那是个人效率工具。
M2你们学了怎么用AI写代码——那是编程工具。
M3今天要学的是：怎么识别一个业务问题，值不值得用AI解决。

这就是「业务分析能力」。

核心问题：什么业务适合用AI？

给大家四个判断标准，我们叫AI场景四要素：

第一，重复性高。
同样的话术反复出现，同样操作反复发生。
比如客服每天回答100遍"课程多少钱"——适合AI。
但如果每次回答都不同，AI反而做不好。

第二，规则明确。
处理这件事有明确规则，或者有足够历史数据让AI学习。
比如"退款要审核订单状态"——规则明确，AI能学会。

第三，数据充足。
有历史记录可供AI学习。
如果这个业务从来没发生过，AI没有学习材料。

第四，价值明显。
解决了能带来明确收益——省钱、增效、降投诉。

四要素都满足，优先做。
满足3个，认真评估。
满足2个以下，暂缓。

接下来我们看五个行业的真实AI案例。
```

### 跟练任务（90min）

```
任务：真实业务挖掘

1. 每组聚焦自己公司的真实业务
2. 用AI场景四要素评估
3. 识别3个AI机会
4. 填写机会分析表

输出：AI机会分析表
```

## M3-2 需求分析（Week 7 Day 2）

### 理论逐字稿（30min）

```
昨天找到了AI机会，今天要把机会变成具体需求。

给大家一个三层拆解公式：

业务问题 → AI需求 → 功能要求

举例：
业务问题：客服每天回复1000条重复问题，占用60%时间
AI需求：需要能自动回复常见问题的AI客服
功能要求：
  - 能识别问题类型（退货/咨询/投诉）
  - 能从知识库检索答案
  - 能生成自然语言回复
  - 能识别无法回答的问题转人工

三层拆解的关键：
业务问题要具体，AI需求要清晰，功能要求要可落地。

接下来用MoSCoW排优先级。
M：必须有——不解决业务跑不下去
S：应该有——解决了更好
C：可以有——锦上添花
W：这次不会有——放到来期

原则：M的功能不超过60%。
因为时间有限，要先保证核心功能能跑通。
```

### 跟练任务（90min）

```
任务：TOP 1 AI机会 → 完整需求文档

1. 用三层拆解法分析TOP 1机会
2. 用MoSCoW排优先级
3. 输出完整的需求文档
```

## M3-3 方案设计（Week 8 Day 1）

### 理论逐字稿（30min）

```
需求明确了，接下来设计AI闭环。

什么是AI闭环？简单说：
用户输入 → AI理解 → AI处理 → 输出结果

设计闭环有三个问题必须回答：

第一个：输入边界
AI能处理什么输入？边界外的输入怎么处理？
要设计兜底方案。

第二个：输出可控
AI生成的回复会不会说错话？
需要加人工审核吗？还是AI直接发？

第三个：异常处理
AI不知道怎么处理时，转人工的触发条件是什么？

三个问题想清楚，方案就完成一半。

再给大家一个工具：用户旅程地图。
从用户发起咨询到问题解决，每个环节标注：
- 是AI处理还是人工处理
- 什么情况下从AI切换到人工
```

### 跟练任务（90min）

```
任务：设计AI闭环

1. 画出完整的用户旅程地图
2. 标注每个环节是AI还是人工
3. 确定转人工触发条件
4. 定义MVP（最核心3个功能）
```

## M3-4 成本测算（Week 8 Day 2）

### 理论逐字稿（30min）

```
方案设计完了，老板一定会问：多少钱？能回本吗？

成本分两类：
1. 开发成本——一次性投入
2. 运营成本——持续支出

ROI公式：
ROI = (年收益 - 年成本) / 年成本 × 100%

回本周期 = 开发成本 / (月均收益 - 月均运营成本)

举例：
开发成本3万
月均运营2000
月均人力节省1万
回本周期 = 30000 / (10000-2000) = 3.75个月

回本周期6个月以内，推荐做。
超过12个月，要谨慎。
超过24个月，慎重。
```

### 跟练任务（90min）

```
任务：ROI测算

1. 开发成本估算（明细）
2. 月均运营成本（Token消耗）
3. 月均收益估算
4. 计算回本周期
5. 判断是否可行
```

## M3-5 方案评审（Week 8 Day 3-5）

### 答辩流程

```
每组20分钟：
1. 方案汇报（10min）
2. 讲师提问（5min）
3. 点评（5min）

必须回答的问题：
1. 你的业务场景是什么？AI能解决吗？
2. AI闭环怎么设计的？
3. ROI是多少？回本周期多长？
```

### 评审标准

| 维度 | 权重 | 要求 |
|------|------|------|
| 场景真实 | 20% | 来自真实业务 |
| 需求明确 | 25% | 问题-方案清晰 |
| 闭环完整 | 25% | 有输入-处理-输出 |
| ROI可行 | 20% | 回本周期<12月 |
| 表达清晰 | 10% | 逻辑清晰 |
