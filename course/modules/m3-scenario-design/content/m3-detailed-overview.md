## M3-1 业务挖掘（Week 7 Day 1）

### 学习目标
- [ ] 能识别适合AI的业务场景
- [ ] 掌握需求访谈方法
- [ ] 能分析AI自动化机会

### 理论内容（30min）

#### 什么业务适合AI

```
AI场景四要素：
□ 重复性高：同样的问题/操作重复发生
□ 规则明确：有明确的处理规则或流程
□ 数据充足：有足够的数据供AI学习
□ 价值明显：解决后能带来明确收益
□ 技术可行：当前AI能力能支持

不适合AI的场景（优先排除）：
□ 需要深度情感理解
□ 涉及法律合规判断
□ 实时性要求极高（毫秒级）
□ 完全无历史数据的全新场景
```

#### 需求访谈五步法

```
第一步：问现状——"你们现在怎么处理这个业务？"
第二步：问痛点——"哪个环节最花时间、最让你烦恼？"
第三步：问数据——"有历史数据吗？用什么系统？"
第四步：问目标——"解决后希望看到什么结果？"
第五步：问约束——"有什么是不能做的？"
```

#### 案例：五个行业的AI机会

| 行业 | 典型场景 | AI能做什么 |
|------|---------|-----------|
| 教育培训 | 课程咨询工单分发 | AI识别意图 → 分发顾问 |
| 电商 | 售后工单处理 | AI分类 → 自动处理 → 复杂转人工 |
| 制造业 | 产线报工与质检 | AI视觉检测 → 异常预警 |
| 医疗 | 预约挂号与分诊 | AI理解症状 → 推荐科室 |
| 金融 | 贷款材料初筛 | AI审核材料 → 完整性检查 → 风险提示 |

### 跟练内容（30min）

```
任务：
1. 访谈小组内成员的业务
2. 用AI场景四要素评估
3. 识别3个AI自动化机会
4. 填写机会分析表

输出：AI机会分析表
```

## M3-2 需求分析（Week 7 Day 2）

### 学习目标
- [ ] 能将业务问题转化为AI需求
- [ ] 能评估需求优先级
- [ ] 能输出需求文档

### 理论内容（30min）

#### 需求三层拆解

```
业务问题：现状中的痛点
   ↓
AI需求：AI能做什么（输入→处理→输出）
   ↓
功能要求：具体的功能点（Must/Should/Could）

示例：
业务问题：客服每天回答1000个重复问题，占用60%时间
AI需求：需要一个能自动回复常见问题的AI客服
功能要求：
  - 能识别用户问题类型（退货/咨询/投诉）
  - 能从知识库检索答案
  - 能生成自然语言回复
  - 能识别无法回答的问题转人工
```

#### MoSCoW法则

| 级别 | 含义 | 要求比例 |
|------|------|---------|
| Must | 必须有——不解决业务跑不下去 | 60% |
| Should | 应该有——解决了更好 | 20% |
| Could | 可以有——锦上添花 | 15% |
| Won't | 这次不会有——放到来期 | 5% |

#### 需求文档规范

```markdown
## AI需求文档 - [项目名称]

### 1. 业务背景
- 行业/部门：
- 业务描述：

### 2. 核心问题
解决了什么问题？

### 3. AI需求列表
| 序号 | AI需求 | 解决的问题 |
|------|--------|-----------|
| 1 | | |
| 2 | | |
| 3 | | |

### 4. 功能优先级（MoSCoW）
| 级别 | 功能 | 说明 |
|------|------|------|
| Must | | 不解决业务跑不下去 |
| Should | | 解决了更好 |

### 5. 边界条件
什么情况AI不处理？什么时候转人工？
```

### 跟练内容（30min）

```
任务：
1. 将访谈需求转化为AI需求
2. 用MoSCoW法则排优先级
3. 输出完整的AI需求文档
```

## M3-3 方案设计（Week 8 Day 1）

### 学习目标
- [ ] 能设计完整的AI闭环
- [ ] 能绘制用户旅程地图
- [ ] 能定义MVP

### 理论内容（30min）

#### AI闭环设计

```
AI闭环 = 输入 → AI处理 → 输出

三个设计问题：
1. 输入边界：AI能处理什么？边界外的怎么办？
2. 输出可控：AI说什么话需要审核？直接发还是人工确认？
3. 异常处理：AI处理不了时，转人工的触发条件是什么？
```

#### 转人工触发条件

```
1. 用户明确说"转人工"
2. AI置信度 < 60%（识别不出来）
3. 连续3次回答错误
4. 用户进入投诉/投诉预警场景
5. 涉及敏感操作（退款/取消订单等）
```

#### 用户旅程地图

```
场景：智能客服

1. 用户发起咨询 → AI识别意图
2. AI查询知识库 → 生成回复
3a. 高置信 → 直接发送
3b. 中置信 → AI回复 + 提示"如需人工请回复0"
3c. 低置信 → 转人工
4. 用户收到回复 → 解决问题/继续追问/转人工
5. 结束后 → AI记录会话质量
```

#### MVP定义

```
MVP = 最核心的3个功能，能跑通闭环即可

定义方法：
1. 列出所有想要的功能
2. 问：如果只能做3个，选哪3个？
3. 确认：这3个能否形成闭环？
```

### 跟练内容（30min）

```
任务：
1. 设计AI客服闭环
2. 绘制用户旅程地图（标注AI/人工处理环节）
3. 确定转人工触发条件
4. 定义MVP（3个核心功能）
```

## M3-4 成本测算（Week 8 Day 2）

### 学习目标
- [ ] 能计算Token消耗
- [ ] 能测算ROI
- [ ] 能做成本收益分析

### 理论内容（30min）

#### 成本类型

```
开发成本（一次性）
- 方案设计费
- 开发实施费
- 测试上线费

运营成本（持续）
- AI调用费（Token消耗）
- 人工维护费
- 系统运维费
```

#### Token消耗计算

```
Token是什么：AI按字数收费，1000 Token ≈ 750汉字

主流模型价格参考：
- GPT-4o：~$15/1M Tokens（输入）
- GPT-3.5-turbo：~$0.5/1M Tokens（输入）

月均消耗估算：
用户量 × 每次对话Token数 × 每月对话次数 = 月Token总量
月Token总量 × 模型单价 = 月AI调用费
```

#### ROI公式

```
ROI = (年收益 - 年成本) / 年成本 × 100%

回本周期 = 开发成本 / (月均收益 - 月均运营成本)

收益类型：
- 人力节省：减少多少人工工时 × 人工成本
- 效率提升：处理速度提升多少
- 转化提升：转化率提升带来的收益
```
