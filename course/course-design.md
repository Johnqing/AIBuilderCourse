# 课程设计思路

> 90天AI Builder实战营的教学设计底层逻辑

---

## 一、设计理念

### 1.1 核心问题

```
传统培训的问题：
- 听完就会，做起来就废
- 内容太多，学完就忘
- 工具太多，不知道用哪个
- 场景太远，无法落地

我们的解决方案：
- 先理论后实践：50%理论打基础，50%实践出成果
- 渐进式学习：从简单到复杂，步步为营
- 作业即产品：产出可演示成果
- 真实场景：公司实际业务
- 能力认证：可衡量的技能证明
```

## 二、理论基础

### 2.1 学习金字塔

```
┌─────────────────────────────────────────────────┐
│  学习方式          │  平均留存率  │  课程应用      │
├─────────────────────────────────────────────────┤
│  听讲              │     5%      │  理论讲解      │
│  阅读              │     10%     │  文档资料      │
│  视听              │     20%     │  视频课程      │
│  演示              │     30%     │  案例展示      │
│  讨论              │     50%     │  小组研讨  ←  │ 
│  实践              │     75%     │  实战工作坊 ←  │ 
│  教授他人          │     90%     │  答辩分享   ←  │
└─────────────────────────────────────────────────┘
```

**我们的课程设计（渐进式）**：
- 理论输入 50%（录播+讲解+演示）
- 实战输出 50%（跟练+实战+项目）
- **新手友好**：先看、先学、先理解，再动手做

### 2.2 刻意练习理论（Deliberate Practice）

| 要素 | 实现方式 |
|------|---------|
| 明确目标 | 每个模块有明确能力目标 |
| 专注投入 | 每日高强度实战 |
| 即时反馈 | 作业点评+导师指导 |
| 持续改进 | 迭代优化 |

### 2.3 成人学习理论（Andragogy）

| 原则 | 应用 |
|------|------|
| 经验导向 | 用真实业务场景做练习 |
| 自主导向 | 可选路径（执行者/管理者/创业者） |
| 问题导向 | 每节解决一个实际问题 |
| 即时应用 | 学完立刻能用 |

## 三、教学模式

### 3.1 ADDIE模型

```
A - Analysis（分析）
  → 学员基础分析、业务场景分析
  
D - Design（设计）
  → 课程目标、教学方法、考核方式
  
D - Develop（开发）
  → 教学内容、练习题、案例
  
I - Implement（实施）
  → 授课、练习、项目
  
E - Evaluate（评估）
  → 考核、反馈、迭代
```

### 3.2 翻转课堂模式

```
传统课堂：
  课上听讲（被动） → 课后作业（无人指导）

翻转课堂：
  课前预习（主动） → 课上练习（导师指导） → 课后复盘（内化）
```

### 3.3 项目制学习（PBL）

```
项目驱动学习：
  真实项目 → 需求分析 → 方案设计 → 开发实施 → 成果展示
  
每个阶段都是学习机会，而非学完再考
```

## 四、课程架构设计

### 4.1 能力模型

```
AI Builder能力 = 产品认知 × Vibe Coding × 商业落地
                     ↓            ↓            ↓
                  理解能力      实现能力       价值能力
```

**三层能力的递进关系**：
1. **理解能力**：知道AI Builder是什么、能做什么
2. **实现能力**：能用AI Builder做出东西
3. **价值能力**：能用AI Builder解决真实业务问题

### 4.2 模块递进设计

```
M1（2周）     M2（4周）     M3（2周）     M4（2周）     M5（2周）
┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
│ AI Builder│  │ Vibe    │  │ 场景    │  │ 商业    │  │ 实战    │
│ 基础     │→ │ Coding  │→ │ 设计    │→ │ 落地    │→ │ 项目    │
│          │  │         │  │         │  │         │  │         │
│ 产品架构  │  │ AI辅助   │  │ 业务挖掘 │  │ 场景落地 │  │ 真实场景 │
│ 技能体系  │  │ 编程    │  │ 需求分析 │  │ 成本测算 │  │ 从0到1  │
│ 工作流    │  │ 原型搭建 │  │ 方案设计 │  │ MVP验证 │  │ 上线答辩 │
└─────────┘  └─────────┘  └─────────┘  └─────────┘  └─────────┘
     ↓            ↓            ↓            ↓            ↓
   L1认证      L2认证       L3认证       方案评审      L4认证
```

### 4.3 学习路径设计

```
入口：能力自测（5题，5分钟）
    ↓
必选：M0预备模块（AI基础认知）
    ↓
选择一条角色路径 ↓
┌────────────┬────────────┬────────────┐
│ 执行者路径  │  管理者路径  │  创业者路径  │
│ (运营/人)  │  (PM/总监)  │  (商业落地)  │
└────────────┴────────────┴────────────┘
    ↓
终局：能力认证测试
    ↓
达标 → 获得「AI Builder实战」认证
```

## 五、每日教学设计

### 5.1 时间分配（新手友好型）

```
┌──────────────────────────────────────────────────┐
│  时间段       │  形式     │  内容        │  占比  │
├──────────────────────────────────────────────────┤
│  09:00-10:30 │  理论讲解  │  概念+原理+演示  │  25%  │
│  10:30-12:00 │  跟练指导  │  讲师带练一步跟  │  25%  │
│  14:00-16:00 │  实战练习  │  独立完成练习   │  30%  │
│  16:00-18:00 │  复盘答疑  │  作业点评+答疑  │  20%  │
│                           理论学习：50%            │
│                           动手实践：50%            │
└──────────────────────────────────────────────────┘
```

**设计原则**：先看、先学、先理解，再动手做。新手不慌。

### 5.2 每周节奏

| 周几 | 形式 | 内容 |
|------|------|------|
| 周一 | 理论学习 | 当周课程理论（录播+直播讲解） |
| 周二 | 跟练 | 讲师带练，每步跟做 |
| 周三-周四 | 实战 | 当天作业独立完成 |
| 周五 | 点评 | 作业点评+答疑 |
| 周六 | 复盘 | 个人复盘30min |
| 周日 | 预习 | 预习下周内容 |

## 六、考核体系设计

### 6.1 过程性考核

```
考核维度 = 输出物（40%） + 能力认证（30%） + 过程表现（20%） + 团队协作（10%）
```

### 6.2 认证体系

| 等级 | 考核方式 | 核心能力 |
|------|---------|---------|
| L1 | 笔试+架构图 | AI Builder操作 |
| L2 | 原型+答辩 | Vibe Coding |
| L3 | 方案评审 | 场景设计 |
| L4 | 项目答辩 | 实战落地 |

### 6.3 考核形式

| 形式 | 作用 |
|------|------|
| 情境测验 | 检验问题分析能力 |
| 实操题 | 检验工具使用能力 |
| 方案文档 | 检验方案设计能力 |
| 答辩路演 | 检验综合表达能力 |

## 七、差异化设计

### 7.1 vs 其他AI课程

| 维度 | 其他课程 | 我们的课程 |
|------|---------|-----------|
| 内容组织 | 技巧罗列 | 能力递进 |
| 学习路径 | 线性无序 | 可选路径 |
| 实践比例 | ≤30% | ≥70% |
| 考核方式 | 纯理论 | 实战+认证 |
| 场景来源 | 虚构场景 | 真实业务 |

### 7.2 vs 传统企业培训

| 维度 | 传统培训 | 我们的课程 |
|------|---------|-----------|
| 学员参与 | 被动听讲 | 主动实战 |
| 成果产出 | 无 | 有可演示成果 |
| 能力验证 | 无 | 认证体系 |
| 落地支持 | 无 | 全程辅导 |

## 八、关键设计原则

### 8.1 做中学原则

```
不要告诉学员怎么做
要让学员自己做出来

讲师角色：
- 不是知识的灌输者
- 是练习的引导者和辅导者
```

### 8.2 作业即产品原则

```
每个作业都要产出可演示的东西

作业评价标准：
- 不是"做得对不对"
- 而是"能不能用"
```

### 8.3 真实场景原则

```
禁止使用虚构场景练习

每个小组必须：
- 用自己公司的真实业务
- 做自己公司的真实项目
```

### 8.4 能力可衡量原则

```
每个能力点都要有明确的考核标准

学员要知道：
- 学完能达到什么水平
- 如何证明自己达到了
```

## 九、风险与应对

| 风险 | 预防措施 |
|------|---------|
| 学员基础差异大 | 入学测试+预备模块 |
| 完成率低 | 每日小目标+导师跟进 |
| 小组搭便车 | 组内互评+淘汰机制 |
| 场景无法落地 | 优先选择公司真实场景 |
| 导师资源不足 | 小组制+定期集中辅导 |

---

## 十、持续迭代机制

```
每次课程结束后：
1. 收集学员反馈（满意度+建议）
2. 分析作业完成情况（哪些地方容易出错）
3. 访谈导师（哪些地方需要加强）
4. 更新教学内容（案例、练习、讲解重点）
5. 更新考核标准（根据实际情况调整）

形成闭环：
设计 → 实施 → 评估 → 改进 → 再设计
```
