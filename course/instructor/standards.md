# 讲师标准与培训体系

## 一、讲师能力标准

### 1.1 基本要求

| 要求项 | 标准 |
|-------|------|
| 技术背景 | 有编程或AI相关经验 |
| AI使用经验 | 日常使用AI工具 |
| 表达能力 | 能清晰讲解概念 |
| 耐心程度 | 愿意解答学员问题 |

### 1.2 分级标准

| 等级 | 要求 | 可授课程 |
|------|------|---------|
| 初级讲师 | L2认证 + 模拟试讲通过 | M0-M1 |
| 中级讲师 | L3认证 + 实际授课10h+ | M0-M2 |
| 高级讲师 | L4认证 + 实际授课50h+ | M0-M4 |
| 专家讲师 | L4认证 + 授课100h+ + 案例开发 | M0-M5 |

## 二、讲师培训体系

### 2.1 培训流程

```
成为讲师流程：

1. 申请条件
   - L2认证通过
   - 通过试讲评估

2. 预备讲师
   - 跟随高级讲师观摩3次
   - 辅助答疑10小时
   - 模拟试讲1次

3. 正式讲师
   - 试讲通过
   - 分配课程模块
   - 开始独立授课

4. 持续提升
   - 定期教研
   - 学员反馈改进
   - 每年重新认证
```

### 2.2 讲师培训内容

| 培训模块 | 内容 | 时长 |
|---------|------|------|
| 课程内容 | 熟悉所有模块内容 | 4小时 |
| 授课技巧 | 讲解方法、节奏把控 | 2小时 |
| 答疑技巧 | 处理学员问题 | 2小时 |
| 工具使用 | OpenClaw、飞书等 | 2小时 |
| 试讲演练 | 模拟授课 | 2小时 |

## 三、讲师质量保证

### 3.1 质量监控

- 每节课结束后收集学员反馈
- 定期抽查授课录音/录像
- 学员评分低于3.5分（满分5分）需改进

### 3.2 讲师激励

| 激励项 | 内容 |
|-------|------|
| 薪资 | 授课费 + 学员评价奖金 |
| 晋升 | 达到标准可晋升等级 |
| 荣誉 | 优秀讲师称号 |
| 资源 |优先使用新课程材料 |

### 3.3 淘汰机制

- 连续3个月学员评分<3.5分
- 出现重大教学事故
- 年度认证未通过

以上情况取消讲师资格

## 四、讲师资源包

每位讲师配备：
- 完整教案
- 随堂练习答案
- 常见问题解答
- 学员反馈汇总
- 授课工具包
