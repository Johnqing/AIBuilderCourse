# 教育培训工单智能分发

> 工单系统 → Webhook触发 → OpenClaw智能分发 → 空闲顾问处理

---

## 案例背景

| 维度 | 信息 |
|------|------|
| 行业 | 教育培训（K12学科辅导/职业培训） |
| 规模 | 中型机构，月均工单 8000+，班主任/课程顾问 20人 |
| 痛点 | 学员问题分配不均、响应慢、重复性问题占用人工 |
| 目标 | 工单智能分发 + AI预处理，让人效提升50% |

---

## 业务痛点分析

### 教育培训工单分布

| 问题类型 | 占比 | 典型问题 |
|---------|------|---------|
| 课程咨询 | 30% | 课程推荐/报名咨询/价格优惠 |
| 课程服务 | 25% | 上课提醒/课时查询/换课申请 |
| 技术问题 | 20% | 平台登录/直播卡顿/作业提交 |
| 投诉反馈 | 15% | 师资投诉/课程质量/服务态度 |
| 其他 | 10% | 合作加盟/发票开具/转介绍 |

### 原有工单流程问题

```
学员提交工单 → 人工客服随机接待 → 分配不均 → 响应滞后 → 学员体验差
                                    ↓
                            平均响应时间 > 45分钟
```

---

## 解决方案：工单智能分发工作流

### 整体架构

```
学员提交工单（多渠道：App/公众号/电话/班主任微信）
    ↓
工单系统创建工单（记录来源、内容、学员信息）
    ↓
Webhook触发OpenClaw（工单ID + 关键信息）
    ↓
┌─────────────────────────────────────┐
│        OpenClaw 智能分发引擎           │
│  1. 工单预分析（AI提取关键信息）         │
│  2. 学员分层（新签/续费/高价值）          │
│  3. 问题分类（咨询/服务/技术/投诉）       │
│  4. 坐席状态查询（谁空闲）             │
│  5. 智能分配（学员分层匹配顾问能力）       │
│  6. 分发执行（回写工单系统 + 通知顾问）    │
└─────────────────────────────────────┘
    ↓
顾问收到通知（微信/企微/飞书）
    ↓
AI预填跟进建议（可一键发送或修改）
    ↓
顾问处理 → 完结工单
```

## 落地效果

### 量化指标（上线3个月后）

| 指标 | 上线前 | 上线后 | 提升 |
|------|-------|-------|------|
| 平均响应时间 | 45分钟 | 3分钟 | 93%↓ |
| 工单分配均匀度 | 偏差50% | 偏差10% | 80%↓ |
| 顾问人效 | 35件/人/天 | 55件/人/天 | 57%↑ |
| 课程咨询转化率 | 18% | 31% | +13pp |
| 学员满意度 | 75% | 92% | +17pp |

### ROI

```
月节省人力成本: 64000元
月增收（转化率提升）: 约390000元
月总收益: ~454000元
投入回收期: <1个月
```
